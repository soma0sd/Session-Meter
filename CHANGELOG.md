# Changelog

이 프로젝트의 주요 변경 사항을 기록합니다. 형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/)를
따르며, 버전은 [SemVer](https://semver.org/lang/ko/)를 따릅니다. 항목은 마이너 버전 단위로 묶습니다.

## [0.4] - 2026-07-22

### Added

- **멀티 서비스 모니터링**: Claude에 더해 **Antigravity(Gemini 구독)** 사용량을 함께 표시. 로그인은
  별도 프로세스로 격리된 전용 창에서 진행되어, 로그인 페이지 문제가 앱 전체에 영향을 주지 않습니다.
  공식 API가 없어 화면 스크래핑 방식이며 실험적으로 제공됩니다.
- **위젯 스타일 창**: 5가지 그래픽 컨셉(수평 바·동심원 링·듀얼 반원·수직 기둥·육각 링)을 각각
  상세/컴팩트로, 총 10종 중에서 **서비스별로** 선택. 실시간 미리보기와 남은/사용 표시 토글 제공.
- **서비스별 개별 위젯**: 로그인한 서비스마다 독립 위젯 창을 띄우고, 제목표시줄에 서비스 이름을 표시.
- **통계 창 서비스 탭**: 로그인한 서비스별 탭으로 통계를 구분해 표시.
- **개발 빌드 분리 실행**: 개발용으로 실행할 때 별도 식별자(`-dev`)를 사용해, 설치된 정식 버전과
  충돌 없이 동시에 실행.
- **위젯 제목 서비스 아이콘**: 각 위젯 제목에 서비스 브랜드 색상의 아이콘을 표시해 서비스를 한눈에 구분.

### Changed

- 위젯 관련 설정을 설정 창에서 새 **위젯 스타일** 창으로 이동. 기존 상세/컴팩트 전환은 10종 스타일
  선택으로 대체.
- 설정 창 계정 섹션을 **서비스별 로그인/로그아웃 목록**으로 개편.
- 트레이 메뉴에서 위젯 표시/숨김 항목을 제거하고, **위젯 스타일 창에서 서비스별로 개별 표시/숨김**하도록 변경.
- 컴팩트 위젯 스타일의 남은 시간 표시를 시계 형식(`H:MM` / `M:SS`)으로 변경.
- 자동 업데이트 확인을 시작 시 1회에서 **10분 주기**로 변경.
- **Antigravity** 서비스를 **Gemini**로 개명(표시명·내부 식별자). 기존 세션·설정·위치·이력은 자동으로 마이그레이션됩니다.
- 설정 계정 목록에 서비스별 **사용자 계정과 구독 플랜을 함께 표시**. Gemini는 로그인 시 Google 계정 이메일을 함께 표기(best-effort).

### Fixed

- **업데이트 후 위젯 위치가 초기화되던 문제 수정**: 자동 업데이트 재시작 직전 각 위젯의 위치를
  저장하도록 해, 업데이트 후에도 위젯이 원래 자리에 복원됩니다.
- **트레이 메뉴에서 종료 항목이 가려지던 문제 수정**: 메뉴 창 크기를 내용에 맞게 조정.
- **자동 업데이트 배포 안정화**: 릴리스를 초안이 아닌 태그 포함 사전 릴리스로 게시해, 업데이트 설치 파일 URL이 버전 태그와 분리되어 내려받기 실패(404)하던 상황을 방지.
- **통계 창에 다른 서비스 탭이 표시되지 않던 문제 수정**: 창을 포커스할 때 로그인 서비스 목록을 다시 읽어, 시작 시점 타이밍으로 누락된 탭(예: Gemini)이 나타나도록 함.
- **업데이트 후 위젯 스타일·갱신 주기 등 설정이 초기화되던 문제 수정**: 구(0.3.x) 형식 설정 파일을 최초 실행 시 현재 형식으로 저장해 매 실행마다 재변환되지 않도록 하고, 설정·위젯 스타일 창이 포커스 시 최신 설정을 다시 읽어 다른 창의 변경을 덮어쓰지 않도록 함.
- **업데이트 후 재부팅하면 이전 버전으로 되돌아가던 문제 수정**: 시작 프로그램(자동 실행) 등록 경로가 실행 파일의 실제 위치와 어긋나 있으면, 재부팅 시 갱신되지 않은 옛 실행 파일이 실행되던 문제를 해결. 앱 시작 시 자동 실행이 켜져 있으면 현재 실행 파일 경로로 다시 등록해 어긋난 경로를 자동으로 교정합니다.
- **업데이트 후 위젯 스타일이 기본값으로 초기화되던 문제 수정**: 설정 저장이 서비스별 위젯 설정 맵을 통째로 교체하던 것을, 요청에 없는 서비스의 저장된 위젯 설정은 보존하는 병합 방식으로 변경. 창이 위젯 설정을 아직 읽어오기 전에 저장하더라도 위젯 스타일이 지워지지 않습니다.
- **업데이트 소식 창이 0.3 버전까지만 표시되던 문제 수정**: GitHub 페치 실패 시 쓰이는 폴백이 손으로 관리하던 옛 사본이라 최신 버전이 누락됐던 것을, 빌드 시 실제 변경 로그를 포함해 항상 설치 버전과 일치하도록 변경. 여러 줄로 이어진 항목이 첫 줄에서 잘리던 문제도 함께 수정.

## [0.3] - 2026-07-19

### Added

- **컴팩트 위젯 레이아웃**: 이번 세션·주간 세션을 도넛 그래프 한 행으로 표시. 설정에서 상세/컴팩트
  전환(기본 상세).
- **구독 정보 표시**: 통계 창에 구독 플랜(예: Claude Max 20x) 표시.
- **이력 차트 시간축**: 사용 이력 그래프 상단에 가로 시간축 추가.
- **세션별 사용 알림 기준**: 이번 세션과 주간 세션의 사용 % 알림 기준을 각각 설정. 알림을 끄면
  하위 옵션도 함께 비활성화.
- **위젯 표시 감시**: 갱신 주기마다 위젯 표시 상태를 점검해 화면 밖으로 사라진 위젯을 자동 복구.

### Fixed

- **업데이트 시 설정·세션이 초기화되던 문제 수정**: 설정을 원자적 파일 쓰기로 저장하고 위젯 위치를
  별도 파일로 분리해, 자동 업데이트 중 종료로 설정 파일이 손상·초기화되던 문제를 해결. 손상된 설정
  파일은 백업 후 기본값으로 복구.
- **위젯이 화면 밖(보이지 않음)으로 남던 문제 수정**: 최소화 좌표 저장을 차단하고, 저장된 위치가
  화면 밖이면 기본 위치로 복구.
- **설치 후 바로가기에 커스텀 아이콘이 표시되지 않던 문제 수정**: 설치 시 셸 아이콘 캐시를 갱신해
  시작 메뉴 바로가기에 앱 아이콘이 바로 표시되도록 함(개명 후 캐시 잔상 포함).
- 통계 이력 그래프를 실제 시간 기준으로 그리고 **1개월(30일) 보기**를 추가(24시간·7일·1개월이 시간
  스케일로 구분되도록 함). 초기화 일정의 남은 시간이 NaN으로 표기되던 문제 수정.

### Changed

- 설정 창의 각 섹션을 카드로 구분. 사용하지 않던 "트레이 창" 옵션 제거.

## [0.2] - 2026-07-18

### Added

- **자동 업데이트**: 새 버전을 시작 시 자동 확인하고, 위젯 아이콘 버튼과 트레이 컨텍스트 메뉴에서
  원클릭으로 업데이트(다운로드·설치 후 재실행).
- **업데이트 소식 창**: 변경 로그를 설정 언어(한국어/영어)에 따라 창으로 표시. GitHub 저장소에서
  가져오며, 오프라인·미게시 시 번들 사본으로 대체.
- 통계 창에 계정 이름과 이메일 표시.

### Fixed

- 트레이 더블클릭으로 통계 창을 열 때 위젯이 깜빡이거나 사라지던 문제 수정(시스템 더블클릭 시간 반영).
- 첫 실행 시 위젯이 바로 표시되도록 함(로그인 전에도).

## [0.1] - 2026-07-18

### Added

- **트레이 아이콘**: 5시간 세션 남은 사용량을 색상으로 표시(녹색/노랑/빨강), 테마 대응. 좌클릭 위젯,
  더블클릭 통계, 우클릭 커스텀 메뉴.
- **데스크톱 위젯**: 5시간·주간 사용량과 초기화 카운트다운, 항상 위·이동 잠금 토글, 투명도 조절,
  통계·설정 바로가기.
- **통계 창**: 전체 사용 버킷, 사용 이력 차트, 소진 예측, 초기화 일정.
- **커스텀 테마 컨텍스트 메뉴**와 데스크톱 알림(80%·95% 사용, 초기화).
- **다크/라이트/시스템 테마**, 한국어·영어 UI(OS 언어 자동 감지), 시작 시 자동 실행, 단일 인스턴스.
- **claude.ai 세션 쿠키 기반 사용량 조회**. 세션은 Windows DPAPI로 암호화 저장.

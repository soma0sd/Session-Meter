# 보안 정책

## 취약점 신고

보안 문제는 **비공개로** 신고해 주십시오. 이 저장소의 **Security** 탭에서
**Report a vulnerability**(Security advisories → Report a vulnerability)를 이용하면 됩니다.
보안 문제는 공개 이슈로 열지 말아 주십시오.

가능하면 영향받는 버전, 플랫폼, 영향 설명, 재현 절차를 함께 적어 주십시오. 신고는 보통 며칠 내에
접수 확인을 목표로 합니다.

## 범위와 위협 모델

SessionMeter는 로그인된 본인 claude.ai 세션을 이용해 사용자 자신의 Claude 사용량을 claude.ai
비공개 웹 API로 읽어 오는 로컬 데스크톱 앱입니다. 관련 위협 모델은 다음과 같습니다.

- **저장소에 포함되는 비밀.** 공개된 소스에는 세션 쿠키·토큰·개인정보가 없어야 합니다. 런타임
  비밀은 저장소 밖에 존재합니다.
- **로컬 공격자.** 이미 같은 OS 사용자로 실행 중인 프로세스·사람, 또는 암호화되지 않은 프로필 백업.
- **적대적/중간자(MITM) 응답.** 변조되거나 위장된 claude.ai 응답.

이 앱은 네트워크 서비스가 아니며 원격 공격 표면이 없습니다. 특권 작업은 모두 Rust 백엔드에서
수행되고, 임베드된 claude.ai 로그인 창은 앱의 IPC로부터 격리되어 있습니다.

## 세션 저장

로그인 시 확보한 claude.ai 세션 쿠키는 OS 애플리케이션 데이터 폴더의 사용자별 파일
(`session.dat`)에 저장됩니다.

- **Windows에서는 저장 시 DPAPI로 암호화**(사용자 범위)되어 오프라인 디스크, 옮겨진 백업, 다른
  사용자 계정에서 복호화할 수 없습니다.
- **macOS/Linux**(best-effort)에서는 현재 사용자 범위 평문 파일이며 OS 시크릿 서비스 연동은
  예정입니다.
- 어느 플랫폼이든 로그인 상태에서 같은 사용자로 실행 중인 프로세스는 활성 세션을 읽을 수 있는데,
  이는 브라우저가 자체 쿠키를 보관하는 방식과 같습니다.

자세한 내용은 [`README.md`](README.md)의 "보안·프라이버시"와
[`docs/DATA_CAPABILITIES.md`](docs/DATA_CAPABILITIES.md)를 참고하십시오. 로그아웃하면 파일이
삭제됩니다.

## 알려진 권고(advisory)

| 권고 | 구성 요소 | 상태 |
| --- | --- | --- |
| [RUSTSEC-2024-0429](https://rustsec.org/advisories/RUSTSEC-2024-0429) | `gtk` / `gdk` / `atk`(gtk3-rs 바인딩) | 주 대상 플랫폼에는 해당 없음 |

`RUSTSEC-2024-0429`는 gtk3-rs 바인딩에 대한 **정보성 "유지보수 중단"** 권고이며 메모리 안전·코드
실행 취약점이 아닙니다. 해당 크레이트는 Linux 전용이고 Tauri의 WebKitGTK 백엔드를 통해 전이적으로만
들어오며, **Windows 주 대상에서는 컴파일되지 않습니다.** 수정은 상류 Tauri에서 이루어지며, 해당
릴리스가 나오면 반영합니다. CI에 `cargo audit`을 추가할 경우, 이 항목은 이 문서를 참조하는 주석과
함께 의도적으로 무시할 수 있습니다.
